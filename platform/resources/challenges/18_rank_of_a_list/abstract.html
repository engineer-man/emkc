Instructions

Suppose each list (containing an odd number of elements) is divided into a number of ranks that equals the index of its middle element. For example:

[12, 67, 9, 12, 9, 5, 8, 7, 3, 2, 5]
is divided into 5 ranks since the index of its middle element (5) is 5.

The ranks are ordered descendingly and each rank consists of two values. These two values can be found by moving one element to the left and one element to the right starting from the middle element. Using the previous example list we would get the following ranks and their values:

Rank 5: (9, 8)
Rank 4: (12, 7)
Rank 3: (9, 3)
Rank 2: (67, 2)
Rank 1: (12, 5)

Your job is to find the largest rank (and its values) at which a combination of summations and/or subtractions can be performed to the product of its values and the product of the values of the ranks after it (if found) and the middle element number resulting in zero.

Taking the previous list into example again while starting from the middle element then performing the operations on Rank 5, then on Rank 5 and Rank 4, then on Rank 5, Rank 4 and Rank 3, etc. We will find that:

5 + 9 * 8
5 - 9 * 8
are not equal to zero, so the rank can never be 5.
Similarly:
5 + 9 * 8 + 12 * 7
5 - 9 * 8 + 12 * 7
5 + 9 * 8 - 12 * 7
5 - 9 * 8 - 12 * 7
are not equal to zero, so the rank can never be 4, and so on.

However,

5 + 9 * 8 + 12 * 7 - 9 * 3 - 67 * 2
is equal to 0, and the final two numbers in the operation (67, 2) are in Rank 2, so our rank would be 2 and our output would be:
2,67,2
(where the first number is the number of the rank, and the two numbers following it are the values of the rank). If there is not rank that satisfies this, then our rank would be 0.

Guidlines

  • The number of elements of the list is always an odd number.
  • When getting the values of a rank, the element on the left side of the middle element preceeds
  • If the rank is 0 (there is no rank), only print out 0.

    • Inputs

    value1
    The list elements separated by spaces.

    Sample Test Cases

    A list that has a rank
    value1 5 5 6 7 8 2 9
    output 1,5,9

    Explanation

    This list consists of 3 ranks which are:
    Rank 3: (6, 8)
    Rank 2: (5, 2)
    Rank 1: (5, 9)
    By performing the operations we can find out that:
    7 + 6 * 8 - 5 * 2 - 5 * 9
    is the combination that gives 0.
    The last two numbers in the operation (5, 9) are in Rank 1. So, our output would be 1,5,9
    A list that doesn't have a rank
    value1 1 2 3
    output 0

    Explanation

    This list only contains one rank
    Rank 1: (2, 3)
    We can see that
    1 + 2 * 3
    1 - 2 * 3
    are not equal to zero and this is the smallest (and only rank), so there can never be a rank for this list; hence we print out 0.
